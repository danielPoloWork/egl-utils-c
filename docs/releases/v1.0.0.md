# d4np-c v1.0.0

**First stable release** — 2026-06-21

`d4np-c` is a dependency-free, high-performance C11 systems utility library: allocators,
containers, concurrency primitives, strings, file I/O, and diagnostics. It links against nothing
but libc and the platform threads library.

## Highlights

- **Complete surface.** All 25 specified functions across seven modules — `core`, `mem`, `ds`,
  `concurrency`, `str`, `io`, `sys` — are implemented and documented.
- **Verified to the enterprise bar.** Cross-platform build matrix (GCC, Clang, Apple Clang,
  MSVC), ASan/UBSan/TSan, Valgrind, ≥80% line coverage, a warning-free Doxygen reference, and a
  strict-C11 broad-reach conformance build.
- **Ready to consume.** `find_package(d4np-c)` → `d4np::d4np`, plus a vcpkg port and a Conan
  recipe.
- **ABI frozen for 1.x.** The public types, signatures, opaque-storage sizes, and enum values are
  stable across the 1.x line under SemVer (see [ADR-0008](../adr/0008-freeze-public-abi-for-1x.md)).

## Compatibility

Language floor **C11**. Supported toolchains: GCC ≥ 11, Clang ≥ 14, Apple Clang ≥ 14, MSVC ≥
19.30. Platforms: Linux x86_64, macOS arm64, Windows x86_64. See the
[compatibility matrix](../development/compatibility.md).

## Install

```bash
cmake -S . -B build -DD4NP_INSTALL=ON
cmake --build build
cmake --install build --prefix /your/prefix
```

```cmake
find_package(d4np-c CONFIG REQUIRED)
target_link_libraries(app PRIVATE d4np::d4np)
```
